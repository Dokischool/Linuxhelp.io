🔹 1. Modul: LAMP Szerverkörnyezet Kiépítése

A vizsgafeladatok alapja egy működő webszerver és adatbázis-háttér biztosítása. Fontos, hogy a telepítések után mindig ellenőrizd a szolgáltatások állapotát!

sudo apt install mariadb-server apache2 -y
Parancselem / Flag Típus Részletes funkció és vizsgajelentőség
sudo Parancs SuperUser DO. Adminisztrátori (root) joggal futtatja a parancsot.
apt install Parancs A Debian/Ubuntu rendszerek csomagkezelő parancsa új szoftverek letöltéséhez.
-y Flag Yes. Automatikusan elfogadja a telepítés során feldobott megerősítést.

Szolgáltatások státuszának verifikálása

sudo systemctl status mariadb sudo systemctl status apache2

PHP és phpMyAdmin integráció

sudo apt install php libapache2-mod-php php-mysql phpmyadmin -y sudo phpenmod mbstring sudo systemctl restart apache2

💡 Vizsga Tipp a phpMyAdminhoz:

A telepítés közben a rendszer megkérdezi, hogy melyik webszerverhez konfigurálja magát automatikusan. A Space billentyűvel rakj csillagot ([*]) az apache2 elé, majd a Tab gombbal menj az [OK]-ra!

🔹 2. Modul: MySQL Adminisztráció és Biztonságos Felhasználókezelés

mysql -u root -p

A fenti paranccsal lépünk be a MySQL parancssorba. Ezután az alábbi SQL utasításokat adjuk ki (pontosan követve a vizsga anyagát):

CREATE USER 'vizsgazo'@'localhost' IDENTIFIED BY 'Abcd1234'; GRANT ALL PRIVILEGES ON . TO 'vizsgazo'@'localhost' WITH GRANT OPTION; FLUSH PRIVILEGES; EXIT;

⚠️ Megjegyzés a szintaxishoz

A vizsgafeladat forrásanyagában a GRANT ALL PRIVILEGES ON . TO parancs szerepel. Ezt a weblap pontosan így tartalmazza. Mindig kövesd az adott vizsgafeladat pontos leírását!

Szolgáltatás újraindítása és IP Cím Lekérdezése

sudo systemctl restart mariadb ip a

🔹 3. Modul: Könyvtárszerkezet Generálás

Mappastruktúrák létrehozására a vizsgán két módszert is alkalmazhatsz. A `mkdir -p` (parents) kapcsoló garantálja, hogy a szülőkönyvtárak is létrejönnek.

A) Gyorsított módszer (Kapcsos zárójelekkel)

sudo mkdir -p Mentök/{Lègi,Vizi} Tüzoltòl/Zàhony/{Kobra365,Vizisiklò824} tree

FONTOS: A vesszők után nem szabad szóközt tenni a kapcsos zárójelek között!

B) Lépésenkénti, biztonságos módszer

Ha nem szeretnéd egy parancsba sűríteni, soronként is kiadhatod (ahogy az egyik vizsgakép is mutatja). Itt figyelj a pontos elnevezésekre (pl. Kobra356 vs 365):

mkdir -p Mentők/Légi mkdir -p Mentők/Vizi mkdir -p Tűzoltók/Záhony/Kobra356 mkdir -p Tűzoltók/Záhony/Vizisikló824 tree

🔹 5. Modul: Fájljogosultságok (chown / chmod)

sudo chown feigela:vizsgazo /home/feigela/Foci sudo chmod 770 /home/feigela/Foci
  • chown [tulajdonos]:[csoport]: Tulajdonjogok átadása.
  • chmod 770: Tulajdonos: 7 (rwx), Csoport: 7 (rwx), Mindenki más: 0 (---).

🔹 6. Modul: Fájlrendszer Formázása (mkfs)

Amikor a szerverhez új merevlemezt vagy partíciót adunk (pl. /dev/sdb1), azt formázni kell, mielőtt adatokat tárolhatnánk rajta. A leggyakoribb és legstabilabb Linux fájlrendszer az ext4.

sudo mkfs -t ext4 /dev/sdb1 sudo mkfs -t ext4 /dev/sdb2
Parancselem / Flag Magyarázat
mkfs Make FileSystem. Ez a parancs hozza létre az üres fájlrendszert az adott blokkeszközön.
-t ext4 Type. Meghatározza a fájlrendszer típusát. Ha a vizsgán megnyomod a dupla Tab gombot a -t után, a Linux listázza a támogatott típusokat (pl. ext2, ext3, ext4, minix).
/dev/sdb1 A célpartíció fizikai elérési útja. Az sd jelenti a lemezt (SCSI/SATA drive), a b jelenti a második fizikai lemezt, az 1 pedig a lemez első partícióját.

🔹 7. Modul: Felhasználók és Csoportok Kezelése (usermod)

Egy rendszerüzemeltetőnek tudnia kell a felhasználókat különböző jogosultsági csoportokba (pl. Osztály) besorolni. Erre a usermod parancsot használjuk.

sudo usermod -a -G Osztály feigela groups groups feigela
Parancselem / Flag Magyarázat
usermod User Modify. Egy meglévő felhasználói fiók tulajdonságainak (pl. csoporttagság, jelszó, lejárat) módosítására szolgál.
-a Append (Hozzáadás). KRITIKUS kapcsoló! Csak a -G kapcsolóval együtt használható. Azt jelzi, hogy a felhasználót hozzáadjuk az új csoporthoz, anélkül, hogy a régi csoportjaiból törölnénk.
-G Osztály Groups. A másodlagos csoport neve (ide kerül be a felhasználó).
groups [felhasználó] Kiírja a képernyőre, hogy az adott felhasználó (pl. feigela) mely csoportoknak a tagja. Ha nem adsz meg nevet, a saját fiókod csoportjait listázza.

⚠️ Vizsgacsapda:

Ha a usermod parancsnál elfelejted az -a (append) kapcsolót, és csak a -G-t használod, a rendszer törli a felhasználót az összes eddigi csoportjából, és csak abba az egybe teszi be, amit most megadtál! Ezzel a vizsgán véletlenül kizárhatod a felhasználót a fontos helyekről (pl. a sudo csoportból).
